# LexVoice 中文说明

[English](README.md) | [简体中文](README.zh-CN.md)

LexVoice 是一个面向 Obsidian 桌面端的录音、转写和 AI 整理插件。它适合把会议、访谈、课程、视频音频、个人口述和临时讨论整理成可继续使用的 Markdown 笔记。

它不是一个云服务，也不自带 API Key。LexVoice 只负责把录音、转写、整理和笔记写入串起来；真正的语音识别服务和大模型服务由你自己配置。

## 适合谁

- 经常开会，希望自动生成会议纪要、决策、待办和后续动作的人
- 做用户访谈、专家访谈、招聘面试或调研访谈的人
- 看 B 站、YouTube、课程、讲座时，希望把音频内容整理成学习笔记的人
- 喜欢用 Obsidian 管理知识库，希望录音和纪要都留在本地库里的人
- 对隐私有要求，希望可以选择本地转写和本地大模型的人

## 使用前需要准备什么

必需：

- Obsidian 桌面版
- 一个语音转文字服务：可以是云端 API，也可以是本地模型服务
- 一个用于保存录音的 Obsidian 库内文件夹
- 一个用于保存转写纪要的 Obsidian 库内文件夹

可选但推荐：

- 一个大模型 API：用于整理纪要、提炼要点、生成待办、翻译和优化自定义提示词
- 一个虚拟音频设备：用于录制线上会议、视频播放或电脑声音
- 一个领域词汇表：用于提高人名、品牌名、项目名、行业术语的识别准确度

费用说明：

- Obsidian 和 LexVoice 插件本身不收取 API 费用。
- 云端语音转写和云端大模型通常由服务商按量计费、赠送额度或阶段性免费，具体以服务商页面为准。
- 本地语音转写和本地大模型可以不产生 API 费用，但需要你自己安装服务，并承担电脑性能、显存、维护成本。
- 录制线上会议或电脑声音时，虚拟音频设备可能需要单独安装；大多数常用工具有免费方案。

## 首次使用：先配置什么

![LexVoice 首次使用配置概览](docs/images/first-use-settings.svg)

第一次打开 LexVoice 设置页时，不需要把所有选项一次性填完。建议按下面顺序配置：

1. **配置语音转写服务（必填）**：这是录音能变成文字的前提。你可以选择云端语音转写服务，也可以使用本地 Whisper、SenseVoice、faster-whisper 等服务。云端通常需要 API Key，本地通常免费但需要你先启动本地服务。
2. **确认录音与纪要保存路径（建议）**：默认录音文件保存在 `LexVoice/录音`，转写纪要保存在 `LexVoice/转写纪要`，领域词汇表保存在 `LexVoice/词汇表.md`。这些路径都可以在设置里改成你自己的文件夹。
3. **配置大模型 LLM API（推荐）**：如果只需要原始转写，可以暂时不配置；如果希望自动生成会议纪要、学习笔记、待办、翻译或优化自定义提示词，就需要配置大模型服务。
4. **选择音频输入方式（按需）**：只录自己的声音时，选择麦克风即可；如果要录线上会议、B 站、YouTube 或电脑播放声音，通常需要先安装 VB-Cable、BlackHole 等虚拟声卡，再在 LexVoice 里选择对应输入。
5. **设置日记概要、词汇表和提示词（优化项）**：日记概要适合把当天会议摘要和待办写入日记；词汇表适合提高人名、品牌、项目名和行业术语识别准确度；提示词适合把不同任务整理成不同风格的笔记。

最小可用配置是：**语音转写服务 + 保存路径**。更好的体验是：再配置 **LLM API + 词汇表 + 自定义提示词**。

## 安装方式

1. 关闭 Obsidian。
2. 将本仓库文件夹放到你的库目录：

   ```
   <你的库>/.obsidian/plugins/lexvoice/
   ```

3. 重新打开 Obsidian。
4. 进入 设置 -> 第三方插件，启用 LexVoice。
5. 打开 LexVoice 设置页，先配置语音转写服务，再开始录音。

不要提交或公开本地的 `data.json`。它可能包含 API Key、服务地址和你的个人设置。

## 基本使用流程

1. 打开 LexVoice 实时面板或悬浮录音胶囊。
2. 选择音频输入。
3. 选择转写提示词，例如学习笔记、工作纪要、访谈、个人笔记等。
4. 开始录音。
5. LexVoice 会保存录音文件，并把转写内容写入 Markdown 笔记。
6. 如果配置了大模型，LexVoice 会根据所选提示词把转写整理成更可读、可复用的纪要。

默认路径：

- 录音文件：`LexVoice/录音`
- 转写纪要：`LexVoice/转写纪要`
- 领域词汇表：`LexVoice/词汇表.md`

这些路径都可以在设置里自定义。

## 主要功能

- 后台录音：可以一边录音一边继续使用 Obsidian
- 分段或流式转写：适合较长会议和连续讨论
- AI 整理：把原始转写整理成会议纪要、学习笔记、访谈纪要或自定义模板输出
- 导入音频：可以对已有音频文件转写并整理
- 重新整理：可以对已有转写笔记重新选择提示词，生成新版纪要
- 日记集成：如果你启用了日记，并且当天已有日记，可以生成今日会议概要和待办
- 纪要翻译：可在 AI 整理阶段处理外语内容或生成双语笔记
- 领域词汇表：按人名、品牌/机构、项目/产品、行业术语、其他专有名词分区管理
- 短录音过滤：可过滤小于 3 秒的误触录音，不创建文件、不发起转写
- 清理功能：可删除时长很短且没有有效转写内容的空条目
- 更新检查：从 LexVoice 官方 GitHub Release 检查插件更新

## 语音转写服务

LexVoice 当前按你配置的服务发送音频。常见选择包括：

- 硅基流动等兼容音频转写接口的服务
- OpenAI 音频转写接口
- 阿里云百炼 / DashScope，通过兼容网关或适配服务接入
- 本地 faster-whisper、Xinference、whisper.cpp 封装服务，或其他 OpenAI 兼容转写服务

注意：不同服务的接口形态、文件上传方式、鉴权方式和计费方式不完全相同。设置页里的“服务地址”“API Key”“模型名称”需要按你选择的服务填写。

## AI 整理和提示词管理

AI 整理是可选功能。没有配置大模型时，LexVoice 仍然可以保存录音和转写文本；配置大模型后，可以获得更完整的整理能力。

内置提示词围绕真实任务设计，例如工作纪要、访谈、个人笔记和学习笔记。你也可以创建自定义提示词，命名后会直接出现在录音、导入和重新整理的选择列表里。

建议先配置好大模型 API，再到 AI 整理页管理自己的提示词。一个好的提示词应该说明：

- 这段内容的使用场景
- 需要优先提取什么信息
- 是否需要待办、决策、风险、问题清单
- 是否需要翻译或双语输出
- 输出应该面向谁继续使用

## 电脑音频、B 站和 YouTube

Obsidian 桌面端不能稳定、跨平台地直接捕获系统声音。如果你想转写线上会议、B 站、YouTube 或课程视频，通常需要安装虚拟音频设备，再在 LexVoice 里选择对应的音频输入。

常见工具：

- Windows：VB-Cable
- macOS：BlackHole
- Linux：PulseAudio 或 PipeWire 的 monitor source

这些工具不是 LexVoice 的内置依赖，需要你自行安装和配置。

Windows 使用 VB-Cable 时要注意命名反直觉：

- 把会议软件、B 站客户端、浏览器或系统声音的输出设备切到 `CABLE Input`。虽然名字叫 Input，但它在 Windows 里是“播放/输出设备”。
- LexVoice 读取的是对应的 `CABLE Output`，也就是这根虚拟线缆在“录制/输入设备”里的另一端。
- 如果切到 `CABLE Input` 后自己听不到声音，打开“控制面板 → 声音 → 录制”，找到 `CABLE Output`，双击进入“侦听”，勾选“侦听此设备”，并在“通过此设备播放”中选择你的真实耳机或扬声器，然后点“应用”。

## 隐私和数据风险

LexVoice 没有自有云端存储，也不会把录音上传到 LexVoice 服务器。录音文件只保存在你选择的本地 Obsidian 库路径。

当你使用云端语音转写或云端大模型时，音频、转写文本、Prompt 和相关上下文会发送给你自己配置的服务商。请根据内容敏感度选择服务。

涉及涉密、隐私、客户资料、医疗、法务、人事、招聘、内部战略等内容时，建议优先使用本地语音转写和本地大模型，不要通过云端 API 处理。

录音前请遵守所在地法律法规，并在必要时征得参与者同意。

更多说明见 [PRIVACY.md](PRIVACY.md)。

## 已知限制

- 仅支持 Obsidian 桌面端。
- 暂不支持 Obsidian 移动端。
- 本地转写需要单独安装和运行本地服务。
- 录制电脑声音或线上会议通常需要虚拟音频设备。
- API Key 保存在本地插件设置中，请不要提交 `data.json`。

## 开发与构建

LexVoice 按 Obsidian 插件的常规工程结构维护：开发入口是 `src/main.ts`，执行 `npm install` 安装依赖，开发时运行 `npm run dev`，发布前运行 `npm run build` 生成 `main.js`。 `main.js` 会继续提交到仓库并放入 Release，因为 Obsidian 实际加载的就是这个发布文件。

## 发布前检查

发布或打包前请确认：

- 没有提交 `data.json`。
- 没有提交个人 API Key、服务地址或真实会议内容。
- 已包含 `manifest.json`、`main.js`、`styles.css`、`README.md`、`README.zh-CN.md`、`LICENSE`、`PRIVACY.md`、`SECURITY.md`、`THIRD_PARTY_NOTICES.md`。
- `manifest.json` 中的插件 id 唯一，版本号符合语义化版本。
- 已在一个干净的测试库里启用并验证插件。
